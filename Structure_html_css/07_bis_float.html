Autrement dit, les éléments succèdent l'ordre ou ils sont déclarés dans le code HTML. De façon simpliste, les éléments de séquençage tels que "h1", "p", ou "ul", "s" affichent logiquement pqr défaut les uns en dessous des autres tandis que les éléments délimitant pour la plupart des portions de textes (mise en iotalique d'un ou pluisieurs mots par ex.) s'affichient cote à cotes

En fait, il existe initialement deux grands groupes principaux de balises: <les balises de rencu CSS "bloc" (block) et Les balises de renduCSS "en-ligne" ("inline") Ces valeurs de rendu visuel coincident généralement par défaut avec le group d'appartenance HTML
Des propriétés CSS d'un élément (display:block, inline, list-item, inline-block, table, table-cell...) découlent des spécificités d'affichage, son positionnement dans le flux.

Il est possible de sortit du flux naturel un ou plusieurs élément pour modififier leur affichage avec du CSS

Nos deux groups d'éléments peuvent etre classés en deux familles Les éléments de type block (h1, p, ul; ol, dl, table, blockquote, etc..) Les éléments de type inline (a, img, strong, abbr...)

De plus élément de type "block" peut contenir d'autres éléments de type block ainsi que des éléments de type inline. A RETENIR
Un élément de type block d=se différencie des éléments de type inline (en ligne) sur différents poins, dont ceux-ci:

  • Il occupe l'entièreté de la largeur de son conteneur
  • Il permet l'attribution de marges verticales
  • Il permet la modification de sa hauteur et de sa largeur
Le premier point est primordial, nous utiliserons abondamment ce comportementpar défaut bien pratique dans la creation de nos mise en page; Notons également que tout élément peut etre "reclassé" dans la famille opposée grace à la propriété display (que nous verrons plus tard.)

=>Le type "inline" Les éléments de rendu "inline" se placent toujours l'un à cote de l'autre afin de rester dans le texte (par exemple la balise "strong", "img", "a", "span"). Par défaut, il n'est pas prévu qu'ils puissent se positionner sur la page (meme si cela est possible), ni de leur donner des dimensions (hauteur, largeur, profondeut) leur taille va etre déterminée par la texte ou l'élément:image qu'ils contiennent. Référence: http://fr.learnlayout.com/toc.html (en français)

La propriété CSS FLOAT permet de faire flotter des éléments cote à cote, donc de modifier le comportement naturellement de type block en comportement inline. A l'origine le float est prévu pour entourer une image avec du texte (paragraphe).
